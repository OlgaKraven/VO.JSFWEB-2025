    Примерные вопросы для зачета с оценкой по дисциплине «Разработка web-приложений на языке JavaScript»

    Структура оценивания

    • Вопрос 1-го типа — 20 баллов
    • Вопрос 2-го типа — 20 баллов
    • Вопрос 3-го типа — 60 баллов

    Задание 1 (вопросы 1-го типа, 20 баллов)

    Ответить на теоретические вопросы, дать определения, классификации, перечисления.

    1. Что означает термин «Web как платформа» и какие основные компоненты в него входят?
    2. В чём состоит модель client–server и как распределяется ответственность между клиентом и сервером?
    3. Что такое browser runtime и какую роль в нём играет JavaScript?
    4. Чем отличается статический сайт от динамического?
    5. В чём принципиальная разница между web-сайтом, web-приложением и SPA?
    6. Какова роль JavaScript во frontend-архитектуре?
    7. Как используется JavaScript на backend-стороне?
    8. Что означает термин Fullstack-разработка?
    9. Для чего применяются библиотеки и фреймворки в экосистеме JavaScript?
    10. Что такое HTTP и какую задачу он решает в web-приложениях?
    11. В чём различие между HTTP и HTTPS?
    12. Что такое HTTP-метод и какие методы используются чаще всего?
    13. Что означают HTTP-статусы ответа?
    14. Как работает модель запрос–ответ?
    15. Почему JSON стал основным форматом обмена данными в web?
    16. Что понимается под REST как архитектурным стилем?
    17. Что такое CORS и зачем он нужен?
    18. Для чего используется среда разработки VS Code?
    19. Какие задачи решают DevTools браузера?
    20. Зачем нужно форматирование кода?
    21. Какую роль играют Prettier и ESLint?
    22. Что такое отладка и зачем она необходима?
    23. Что такое ECMAScript и как он связан с JavaScript?
    24. Для чего используется директива use strict?
    25. Какие основные типы данных существуют в JavaScript?
    26. В чём различие между let и const?
    27. Что такое область видимости переменных?
    28. Что понимается под неявным преобразованием типов?
    29. Какие основные управляющие конструкции используются в JavaScript?
    30. Что такое функция и какую роль она играет в структуре программы?

    Развёрнутые аналитические вопросы с пояснениями и примерами.

    2. В каких случаях использование SPA оправдано, а в каких — избыточно?
    3. Почему браузер не может напрямую работать с базой данных?

    6. Почему ошибка CORS проявляется в браузере, но не на сервере?
    7. Как HTTP-статусы помогают разработчику в отладке приложения?
    8. Чем опасна передача данных без HTTPS?
    9. Почему REST не является стандартом, но широко используется?
    10. В каких случаях JSON может быть неудачным форматом обмена?
    11. Почему «рабочий» код может считаться плохим с точки зрения качества?
    12. Какие риски возникают при отсутствии форматирования кода в команде?
    13. Почему умение читать сообщения об ошибках важнее, чем запоминание синтаксиса?
    14. Как отладчик помогает понять логику выполнения программы?
    15. Почему автоматические инструменты не заменяют понимание кода?
    16. Почему неявные преобразования типов считаются источником ошибок?
    17. В каких ситуациях использование const предпочтительнее let?
    18. Почему понимание приоритета операторов важно для корректной логики?
    19. Как use strict влияет на надёжность кода?
    20. Почему даже простое взаимодействие с пользователем требует проверки данных?
    21. Почему сложные бизнес-правила нельзя реализовать без условий и циклов?
    22. Как пошаговый разбор кода помогает находить логические ошибки?

    26. Почему массив объектов часто сравнивают с таблицей сущностей информационной системы?
    27. Как асинхронность влияет на восприятие интерфейса пользователем?
    28. Почему обработка ошибок сети является частью бизнес-логики?
    29. В чём преимущество делегирования событий по сравнению с прямыми обработчиками?
    30. Почему качество кода и валидация форм критичны для реальных web-приложений?

    Задание 3 (вопросы 3-го типа, 60 баллов)

    Практико-ориентированные задания, требующие анализа, проектирования и разработки кода.

    Задание 1.

    Кейс

    В рамках web-приложения реализуется модуль первичной проверки числовых данных.
    Система получает и хранит массив чисел, сформированный из пользовательского ввода, файла или автоматически сгенерированных значений. Для последующей аналитической обработки требуется определить минимальное корректное значение, удовлетворяющее заданному условию.

    В качестве критерия корректности в данном кейсе используется принадлежность числа к чётным значениям, что позволяет отработать навыки фильтрации данных и базовой алгоритмической обработки массива.

    Задание

    Разработайте программу на языке JavaScript, которая:

    1. принимает одномерный массив целых чисел;
    2. последовательно анализирует элементы массива;
    3. отбирает только чётные значения;
    4. определяет минимальное чётное число среди отобранных элементов;
    5. корректно обрабатывает ситуацию, при которой в массиве отсутствуют чётные числа.

    Алгоритм должен быть реализован с использованием базовых конструкций языка JavaScript
    (условные операторы, циклы или методы работы с массивами).

    Результат

    • если в массиве присутствуют чётные числа — в консоль выводится минимальное чётное значение;
    • если чётные числа отсутствуют — в консоль выводится сообщение:
      «Чётные числа не найдены».

    Дополнительно фиксируется корректность работы программы на различных наборах данных
    (пустой массив, массив с одним элементом, массив с отрицательными числами).

    Задание 2.

    Кейс

    В разрабатываемом web-приложении реализуется модуль первичной обработки пользовательских данных.
    Одним из обязательных параметров является возраст пользователя, который используется для логической классификации и последующей настройки поведения системы (доступ к функциям, отображение интерфейса, аналитика).

    Возраст вводится пользователем в явном виде и требует проверки корректности и отнесения к одной из заранее определённых категорий.

    Задание

    Разработайте программу на языке JavaScript, которая:

    1. принимает значение возраста, введённое пользователем;
    2. проверяет корректность введённых данных (числовое значение, неотрицательное);
    3. на основе значения возраста определяет категорию пользователя:
    4. ребёнок — возраст до 18 лет;
    5. взрослый — возраст от 18 до 64 лет включительно;
    6. пенсионер — возраст 65 лет и старше;
    7. реализует логику проверки с использованием условных операторов.

    Результат

    • в консоль выводится категория пользователя в соответствии с введённым возрастом;
    • при некорректном вводе (отрицательное число, нечисловое значение) выводится диагностическое сообщение об ошибке;
    • логика корректно обрабатывает граничные значения (18 и 65 лет).

    Задание 3.

    Кейс

    В информационной системе учёта заказов формируется массив данных, содержащий суммы оформленных заказов.
    Данный массив используется для расчёта финансовых показателей: выручки за период, контроля корректности данных и подготовки аналитических отчётов.

    Каждый элемент массива представляет собой числовое значение — сумму отдельного заказа. Для получения агрегированного показателя требуется вычислить общую сумму всех заказов.

    Задание

    Разработайте программу на языке JavaScript, которая:

    1. принимает одномерный массив числовых значений, представляющих суммы заказов;
    2. последовательно обрабатывает элементы массива;
    3. выполняет суммирование всех значений;
    4. корректно обрабатывает особые случаи:
    5. пустой массив;
    6. массив с одним заказом;
    7. наличие нулевых значений.

    При реализации используйте базовые конструкции языка JavaScript
    (циклы или методы работы с массивами).

    Результат

    • в консоль выводится итоговая сумма всех заказов;
    • при отсутствии заказов выводится значение 0 либо диагностическое сообщение о пустом массиве;
    • корректность расчёта подтверждается на нескольких тестовых наборах данных.

    Задание 4.

    Кейс

    В web-приложении реализуется модуль анализа текстовых данных, поступающих от пользователя.
    Строка может представлять собой имя, комментарий, сообщение формы обратной связи или произвольный текст. Для предварительной обработки и анализа требуется определить количество гласных букв, содержащихся во введённой строке.

    Подсчёт гласных используется как упрощённый пример работы со строками, символами и условиями, а также для демонстрации алгоритмов посимвольного анализа текста.

    Задание

    Разработайте программу на языке JavaScript, которая:

    1. принимает строку, введённую пользователем;
    2. выполняет посимвольный анализ строки;
    3. определяет, является ли каждый символ гласной буквой;
    4. учитывает как строчные, так и прописные буквы;
    5. корректно обрабатывает особые случаи:
    6. пустая строка;
    7. строка без гласных;
    8. наличие пробелов, цифр и специальных символов.

    Для реализации используйте базовые средства языка JavaScript
    (циклы, условные операторы, методы работы со строками).

    Результат

    • в консоль выводится количество гласных букв, найденных в строке;
    • при отсутствии гласных выводится значение 0;
    • программа корректно работает независимо от регистра символов.

    Задание 5.

    Кейс

    В информационной системе управления пользователями хранится список учётных записей.
    Каждый пользователь описывается набором свойств, включая идентификатор, имя и роль в системе (например: user, admin, moderator). Роль определяет уровень доступа и набор доступных функций.

    Для выполнения административных операций и контроля прав доступа требуется получить отдельный список пользователей, обладающих ролью администратора.

    Задание

    Разработайте программу на языке JavaScript, которая:

    1. принимает массив объектов пользователей;
    2. анализирует свойство, определяющее роль пользователя;
    3. отбирает только тех пользователей, у которых роль равна admin;
    4. формирует новый массив, содержащий только пользователей-администраторов;
    5. корректно обрабатывает особые случаи:
    6. отсутствие пользователей с ролью admin;
    7. пустой массив пользователей;
    8. наличие нескольких пользователей с разными ролями.

    Для реализации используйте базовые конструкции языка JavaScript
    (циклы или методы обработки массивов).

    Результат

    • в консоль выводится массив пользователей с ролью admin;
    • если администраторы отсутствуют, выводится пустой массив либо диагностическое сообщение;
    • исходный массив пользователей не изменяется.

    Задание 6.

    Кейс
